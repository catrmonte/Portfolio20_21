CATATHON

Game Engine Lab

This project implemented Unity tutorial to familiarize self with basics of Unity and updating and loops. Followed a tutorial by Brackeys. Added implementation of kinematic seeking and fleeing as demonstrated in pseudo code by Ian Millington. Two additional levels were also created, with their own distinct color scheme. You can play it here or visit the scripts here.

Command Log

Game Engine Lab

Modified the Catathon project to create a command log by the use of an invoker. Commands to move left or right are sent through the invoker. Rather than have a single movePlayer script, there is one for each movement direction, i.e., moveLeft and moveRight. Commands are then displayed on the player’s screen. You can play the game here, or see the scripts here.

Observer Patter

Game Engine Lab

An implementation of an observer pattern into Catathon. Modifying the score script, text is displayed to the player depending on their progress through the level or if they hit an enemy. The specialText script demonstrates the observer, subscribing as a listener to player’s progress through the level and their collisions. You can play the game here or visit the scripts here.

Finite State Machine

Game Engine Lab

In this demonstration, an implementation of Nystrom’s Heroine was created, demonstrating a finite state machine. Each state is created in its own script and has distinct other states it may transition to. For example, a standing player may move to a crouching, jumping, or stone state. It can then from those states transition into more states, but there’s a specific pattern in which they can change. For example, a player cannot go directly from the standing to diving state, but must first go from standing to jumping and then to diving. You can play the game here or access the scripts here.

Abstract Factory & Decorator

Game Engine Lab

In this project, a combination of an abstract factory and a decorator are presented. Players can press a key to get a certain combo of pasta and toppings, calling the factory to make whichever type of pasta is called for by the player’s input. The actual creation of the pasta is deferred to the decorator, who builds the pasta and adds the necessary other toppings, and calculates the “tastiness” of the pasta, which is printed out to screen. You can play the game here or see the scripts here.

Project 8

This project takes what we learned and implements it in a new game engine Unreal.
